# Gemini CLI

`Gemini CLI` 的吸引力在於，它把 Google 的模型能力直接帶進 terminal 工作流裡。

對習慣在 command line 裡工作的人來說，這種工具的價值很高，因為你不用每次都跳去網頁介面，很多事情可以直接在目前的操作環境裡完成。

## 它適合的工作模式

我會把 Gemini CLI 看成比較適合這些情境：

- 快速問問題
- 生成小段內容
- 幫忙整理想法
- 做一些和開發流程貼近的輔助工作

如果你平常就常在 terminal 讀檔案、跑指令、切 repo，那 CLI 型工具通常會很順手。

## 它的定位

它不是單純把聊天視窗搬到終端機，而是讓模型能力更容易接進你原本的工作節奏裡。

所以它的重點不只是「能不能聊」，而是：

- 能不能少切工具
- 能不能更貼近目前上下文
- 能不能變成日常 workflow 的一部分

## 我會怎麼用它

如果需求是偏快速、局部、輕量的 AI 協助，我會覺得 CLI 很好用。

尤其是當你已經在 command line 裡工作時，Gemini CLI 這種形態就很自然。
