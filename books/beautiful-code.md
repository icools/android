# Beautiful Code

`Beautiful Code` 這本書和 `Clean Code` 的氣質不太一樣。

如果說 `Clean Code` 比較像是在講整潔與基本功，那 `Beautiful Code` 比較像是在問：

好的程式碼，除了整齊之外，還有沒有更深一層的思考與取捨？

## 我為什麼覺得它值得看

因為它不是只在講教條，而是透過不同高手的案例，讓你看到：

- 同樣是寫 code，高手看問題的角度有什麼不同
- 什麼叫做真正有判斷力的設計
- 為什麼有些程式看起來樸素，卻很有力量

## 它的價值不只是技巧

這本書比較像是在培養一種眼光。

你會慢慢感覺到，漂亮的 code 不一定是炫技，而可能是：

- 問題被理解得很清楚
- 結構非常貼合需求
- 有做出剛剛好的取捨
- 沒有多餘的複雜度

## 我覺得適合誰

- 已經有一些開發經驗的人
- 想提升設計眼光的人
- 想從案例中學思考，而不是只背規則的人

## 我的簡單結論

如果 `Clean Code` 幫你建立的是整潔感，`Beautiful Code` 更像是在幫你建立審美與判斷力。

這種東西沒有辦法靠死背長出來，但透過好的案例，會慢慢開始有感。
