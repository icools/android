# Clean Code

`Clean Code` 幾乎算是很多工程師都一定會碰到的一本經典書。

它最核心的價值，不是在教你什麼新潮框架，而是在提醒你：

程式碼不只是能跑就好，還要讓人看得懂、改得動、維護得下去。

## 我覺得這本書最有感的地方

它會一直把你拉回一些基本但很容易被忽略的事情：

- 命名到底清不清楚
- 函式是不是太肥
- 責任有沒有混在一起
- 註解是不是在幫壞 code 擦屁股
- 重複邏輯是不是其實已經在求救

這些東西平常很容易被說成只是風格。

但其實它們常常直接影響之後的維護成本。

## 我覺得適合誰

- 已經會寫 code，但想提升可讀性的人
- 常常被 code review 指出結構問題的人
- 想養成比較穩定 coding 習慣的人

## 我怎麼看它的限制

這本書很值得看，但也不要把它當成宗教經典。

有些原則如果過度僵化地套用，反而會讓程式寫得很刻意。

所以我自己會把它看成：

- 提升敏感度的書
- 幫你建立判斷的書
- 不是每條都要機械照抄的書

## 我的簡單結論

如果你開始意識到「我不只是想把功能做出來，我還想把 code 寫得比較像樣」，那 `Clean Code` 幾乎一定值得放進你的閱讀清單。
