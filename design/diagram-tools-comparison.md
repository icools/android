# **工程師畫圖神器大亂鬥：Mermaid vs Draw.io vs 傳統 UML**

平常看 Code 已經夠傷眼了，遇到那種邏輯繞來繞去的「祖傳屎山」，不畫個架構圖真的會通靈到走火入魔。但是！拜託不要再用 PowerPoint 或 Word 裡面的形狀在那邊慢慢拉線了，那是給麻瓜用的。  
身為專業的軟體工程師，畫圖講求的是**效率、版控、可維護性**。這裡直接幫你把市面上的主流畫圖工具扒光檢視，實話實說，告訴你什麼情境該用什麼工具！

## **1\. Mermaid：現代工程師的標配，AI 的最佳損友 🧜‍♀️**

如果你還沒用過 Mermaid，現在、立刻、馬上給我去學！它是基於 JavaScript 的圖表繪製工具，主打「**用寫 Code 的方式畫圖 (Docs as Code)**」。

* **優點 (爽點)：**  
  * **純文字操作：** 語法極度簡潔，手不用離開鍵盤就能畫完一張循序圖。  
  * **版控神器：** 因為是純文字，進 Git 版控超完美，誰改了哪一條線一清二楚，再也沒有合併衝突的 XML 噩夢。  
  * **生態系王者：** GitHub、GitLab、Notion、Obsidian 原生支援！貼上去直接渲染成圖，潮到出水。  
  * **AI 最愛：** 把架構描述丟給 ChatGPT 或 Gemini，請它「輸出 Mermaid 語法」，連畫圖都不用自己動手了，直接實現全自動化。  
* **缺點 (痛點)：**  
  * **排版偶爾起乩：** 它的排版是引擎自動計算的。有時候你想把某個框框往左挪 10 pixel？抱歉，辦不到。節點一多，線條可能會交錯到讓你懷疑人生。  
* **適用場景：** 日常開發紀錄、寫 README、串接 AI 自動產圖。

## **2\. Draw.io (diagrams.net)：所見即所得的可靠老黃牛 🐂**

歷史悠久、免費、無腦好用。就是一個給你一堆形狀和線條，讓你在畫布上自己拉拉拖拖的工具。

* **優點 (爽點)：**  
  * **排版控制狂的最愛：** 你想要這個框框對齊那條線？想要漸層填色？想要放個自訂的 Icon？通通隨便你拉。像素級別的完美掌控。  
  * **無腦上手：** 給 PM、SA 甚至業務看，他們也能輕易接手修改。  
  * **雲端整合：** 直接存在 Google Drive 或 OneDrive，協作很方便。  
* **缺點 (痛點)：**  
  * **手刻地獄：** 每次新增一個節點，你就要手動調整其他十個節點的位置，牽一髮動全身，浪費生命。  
  * **版控災難：** 雖然它可以存成 .drawio 或 XML，但你在 Git 裡面看那些 Diff 根本是天書，完全不知道圖到底改了什麼。  
* **適用場景：** 畫給老闆或客戶看的「火力展示用」精美架構圖、需要高度客製化排版的系統圖。

## **3\. PlantUML：Mermaid 的老前輩，強大但偏笨重 🦕**

在 Mermaid 崛起之前，PlantUML 是 Docs as Code 的霸主。一樣是純文字畫圖。

* **優點 (爽點)：**  
  * **功能爆幹強大：** 支援的 UML 種類和圖表樣式比 Mermaid 多非常多，你要畫多複雜、多嚴謹的圖它都有支援。  
* **缺點 (痛點)：**  
  * **環境配置麻煩：** 底層依賴 Java 跟 Graphviz。光是叫你裝個 Java 環境，很多現代前端工程師就直接放棄了。  
  * **語法稍微囉嗦：** 比起 Mermaid 的輕量級語法，PlantUML 寫起來比較「老派」。  
* **適用場景：** 老專案的延續、需要畫極度嚴格且複雜的標準 UML 圖。

## **4\. 傳統 UML 巨獸 (StarUML, Enterprise Architect) 🦖**

這種工具通常是買斷制或訂閱制，打開介面會有一種回到 Windows XP 時代的懷舊感。

* **優點 (爽點)：**  
  * **絕對的標準化：** 嚴格遵守 UML 規範，什麼實作、繼承、聚合、組合的箭頭絕對不會讓你畫錯。  
  * **雙向工程：** 理論上可以從圖生 Code，或從 Code 生圖（但通常生出來的東西你還要修半天）。  
* **缺點 (痛點)：**  
  * **拿大砲打小鳥：** 肥大、笨重、超級貴。為了解釋一個簡單的 API 流程開這種軟體，簡直是殺雞用牛刀。  
  * **學習曲線陡峭：** 工具本身比 UML 還難學。  
* **適用場景：** 國防工業、金融業核心系統、或是那些被合約綁死「必須交付標準 UML 軟體工程文件」的傳統大廠專案。能躲就躲。

## **5\. 同場加映：Excalidraw —— 裝逼與靈感的文青畫板 🎨**

這兩年紅到發紫的工具。主打「手繪風格」。

* **優點 (爽點)：**  
  * **降低焦慮感：** 因為畫出來都是手繪歪歪扭扭的線條，你再也不會去糾結「這條線有沒有對齊」、「顏色好不好看」。專注在**邏輯跟靈感**本身。  
  * **極致順暢：** 快捷鍵超好用，跟 Obsidian 整合起來簡直是神兵利器（Excalidraw plugin）。  
* **缺點 (痛點)：**  
  * 不能算嚴謹的 UML 工具，畫太複雜的架構圖會像是一團塗鴉。  
* **適用場景：** 敏捷開發的 Brainstorming、跟同事快速討論系統架構、寫部落格文章放幾張裝逼的手繪圖。

## **總結：不要瞎忙，選對工具準時下班**

| 工具名稱 | 核心概念 | 版控 (Git) 友善度 | AI 產圖難易度 | 推薦指數 | 一句話點評 |
| :---- | :---- | :---- | :---- | :---- | :---- |
| **Mermaid** | 純文字生圖 (Docs as Code) | 🟢 極佳 | 🟢 極佳 (無腦產出) | ⭐⭐⭐⭐⭐ | 現代工程師標準配備，必學！ |
| **Draw.io** | 手動拉框框 (WYSIWYG) | 🔴 災難 | 🔴 極難 | ⭐⭐⭐⭐ | 給麻瓜、長官看圖的最佳解。 |
| **Excalidraw** | 手繪白板 | 🟡 普通 | 🔴 難 | ⭐⭐⭐⭐ | 靈感爆發、快速討論的裝逼神器。 |
| **PlantUML** | 純文字生圖 (老派) | 🟢 極佳 | 🟡 普通 | ⭐⭐⭐ | 功能強但環境煩，Mermaid 的備胎。 |
| **傳統 UML 工具** | 嚴謹的軟體工程學 | 🔴 糟透了 | 🔴 不可能 | ⭐ | 拜託不要，你的青春不該浪費在這裡。 |

**最終結論：** **平日開發自己看、寫文件、請 AI 幫忙整理架構，唯一死忠支持 Mermaid。** **如果要放進精美簡報騙投資人或安撫高層，乖乖打開 Draw.io 慢慢刻。**
